Memory Tagging (MTE) Support

For devices supporting it, Dankium is built to leverage **Memory Tagging Extension (MTE)**. To fully benefit from MTE (especially in **SYNC mode** for enhanced exploit detection), it must be enabled on your device. This can typically be done in Android's developer settings, or for devices running Android 16 (released June 2025) or later by enrolling in the Advanced Protection Program.

If you are running GrapheneOS on a supported device, MTE can be enabled via its dedicated security settings. Please note that GrapheneOS implements MTE as **asymmetric memory tagging**, where SYNC mode primarily applies to memory **reads**, while on stock Android, SYNC mode applies to both reads and writes. When active, MTE offers substantial protection against undiscovered security vulnerabilities by immediately detecting memory access violations, providing enhanced defense against even novel exploits.

Comprehensive Mitigation

Even without memory tagging, Dankium employs **robust mitigations against common flaws**. We actively disable just-in-time (JIT) JavaScript, enable shadow call stacks, and integrate other critical compiler mitigations like Control Flow Integrity (CFI). This multi-layered approach means at least **50% of JavaScript-related flaws simply won't work on Dankium**.
